# 1️⃣ Project Overview

Project name:

```
syswatch
```

A **system monitoring CLI tool written in Rust**.

Purpose:

* Monitor system health
* Inspect processes
* Export metrics
* Work like a **lightweight node exporter**

Think of it as a **mini DevOps monitoring agent**.

Inspired by tools like:

* htop
* Prometheus Node Exporter
* Glances

But much simpler.

# 4️⃣ Project Architecture

We will follow a **real backend service structure**.

```
syswatch/
│
├─ Cargo.toml
│
└─ src/
    │
    ├─ main.rs
    │
    ├─ config/
    │   └─ mod.rs
    │
    ├─ models/
    │   └─ system_stats.rs
    │
    ├─ services/
    │   └─ system_collector.rs
    │
    ├─ handlers/
    │   └─ cli_handler.rs
    │
    ├─ utils/
    │   └─ formatter.rs
    │
    └─ errors/
        └─ system_error.rs
```

Let's understand each folder.

---

# main.rs

Entry point.

Responsibilities:

```
start program
load config
call CLI handler
```

---

# config/

Future configuration handling.

Example:

```
refresh rate
log level
metrics port
```

---

# models/

Defines **data structures**.

Example:

```
SystemStats
ProcessInfo
DiskStats
```

Example struct:

```rust
pub struct SystemStats {
    pub cpu_usage: f32,
    pub memory_used: u64,
    pub memory_total: u64,
}
```

Models only store **data**.

---

# services/

Business logic.

Example:

```
collect system stats
inspect processes
read /proc
```

Example:

```
system_collector.rs
```

This module gathers system info.

---

# handlers/

User command handling.

Example:

```
syswatch
syswatch watch
syswatch top
```

CLI logic lives here.

---

# utils/

Helper utilities.

Example:

```
format memory nicely
convert uptime
logging helpers
```

---

# errors/

Custom error types.

Example:

```
SystemError
FileReadError
ParseError
```

This is where we will learn:

```
Result<T,E>
thiserror
anyhow
```
